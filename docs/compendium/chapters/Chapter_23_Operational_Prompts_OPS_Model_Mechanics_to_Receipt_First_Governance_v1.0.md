## 23.3 The Six Operational Prompts (Doctorantura Standard)

These prompts represent six minimal, high-value governance translations of model mechanics into auditable enforcement.

### 23.3.1 Probabilistic Distribution Audit Prompt

**Mandate:**  
For high-risk or high-stakes outputs, log not only the selected output, but the top-N candidate outputs with their associated probability mass.

**TruthSeal purpose:**  
Captures near-miss risk and demonstrates that the system considered, then rejected, unsafe alternatives under the active mandate.

**Minimum receipts:**  
1) Top-N candidate set with probabilities  
2) Policy and mandate version  
3) Model version and integrity hash  
4) Trigger reason (risk domain)  
5) Decision log index

**Failure consequence:**  
Missing distribution logging in a governed domain triggers immediate audit restriction and may assign a Consequence Tier.

---

### 23.3.2 Parameter Hash Validation Prompt

**Mandate:**  
A validated model state must publish a cryptographic integrity hash that is anchored as a governance receipt.

**TruthSeal purpose:**  
Prevents silent weight drift and establishes a stable identity for lawful deployment epochs.

**Minimum receipts:**  
1) Model hash  
2) Validation epoch identifier  
3) Guardian sign-off record  
4) EVL reference pointer  
5) Cross-link eligibility for QENC

**Failure consequence:**  
Unexplained hash changes in a governed deployment trigger Tier escalation and suspension until re-certified.

---

### 23.3.3 Training Lineage Trace Prompt

**Mandate:**  
For training and fine-tuning stages used in governed systems, require data provenance and lineage receipts sufficient for independent reconstruction.

**TruthSeal purpose:**  
Allows bias, contamination, or unsafe policy drift to be traced to training sources with evidentiary clarity.

**Minimum receipts:**  
1) Approved dataset registry references  
2) Source classification and license basis  
3) Sampling and filtering records  
4) Training epoch mapping  
5) Audit-ready lineage summary

**Failure consequence:**  
Unverifiable lineage in a high-risk domain is treated as a critical governance deficit.

---

### 23.3.4 Contextual Attention Isolation Prompt

**Mandate:**  
When an output violates the Moral Impact Forecast or Intent Integrity Gates, record the minimal attention or influence trace sufficient for forensic review.

**TruthSeal purpose:**  
Enables root-cause accountability without relying on narrative explanations alone.

**Minimum receipts:**  
1) Triggered gate identifier  
2) Input segment influence trace (top-K)  
3) Output risk classification  
4) Remediation action record  
5) EVL event pointer

**Failure consequence:**  
Repeated violations without trace capture escalate to stronger operational restriction.

---

### 23.3.5 Non-Compensatory Logic Prompt

**Mandate:**  
When explanatory depth falls below required audit thresholds in governed domains, default to consequence-first enforcement.

**TruthSeal purpose:**  
Prevents “opacity as a loophole.”  
Lack of explanation does not reduce responsibility.

**Minimum receipts:**  
1) Explanation threshold rule  
2) Observed explanation deficit record  
3) Outcome risk tag  
4) Consequence Tier assignment when applicable  
5) CHS band impact record

**Failure consequence:**  
Opaque high-stakes decisions without adequate fallback enforcement are treated as critical breaches.

---

### 23.3.6 Moral Reward Alignment Prompt

**Mandate:**  
Alignment and reward loops must prioritise the active sovereign purpose and Moral Impact Forecast above engagement or preference optimization.

**TruthSeal purpose:**  
Ensures the system optimises for lawful, bounded authority rather than unconstrained utility.

**Minimum receipts:**  
1) Reward objective declaration  
2) SIS-linked purpose binding reference  
3) Training/finetune policy identifier  
4) Evaluation evidence for alignment stability  
5) Independent Guardian attestation record

**Failure consequence:**  
Reward-objective mismatch with the active mandate triggers immediate remedial gating.
